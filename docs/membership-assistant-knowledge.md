# 会员体系 AI 小助手知识库

本文档用于上传给 AI 小助手，帮助它回答用户关于会员等级、升级条件、签到奖励、折扣权益、Claude 权益和异常处理的问题。

回答用户时，请优先使用“用户可理解”的说法；涉及后台实现、分组名、配置项时，只在管理员或内部排查场景下说明。

## 核心口径

- 会员身份以账号当前分组为准，前台展示为：普通用户、Standard 优、Pro优、Super优、Ultra优。
- `Standard` 是正确展示名，不建议写成 `Stand 优`；系统识别上可兼容 `stand` / `standard` 相关分组名。
- 升级判断依赖“有过充值记录 + 累计余额消耗达到门槛”。刚充值但尚未达到消耗门槛时，通常不会立即升级。
- 升级由自动任务或外部项目调整用户分组，本站前台负责展示当前身份，并按分组执行签到、折扣和模型权限。
- 签到奖励每天一次，按用户当前会员分组在配置区间内随机发放。
- 会员权益、模型权限和签到额度如果与本文档不一致，以后台实际配置和用户当前可见模型列表为准。
- AI 小助手不能承诺人工升级、人工补签、手动改余额、退款或绕过分组权限。

## 签到奖励配置

当前推荐签到梯度如下。这里按站内常用换算说明：`500000 quota = 1.0` 显示额度，因此 0.1 - 0.3 对应 `50000 - 150000`。

```json
{
  "default": { "min_quota": 50000, "max_quota": 150000 },
  "standard优": { "min_quota": 150000, "max_quota": 400000 },
  "pro优": { "min_quota": 350000, "max_quota": 600000 },
  "super优": { "min_quota": 450000, "max_quota": 700000 },
  "ultra优": { "min_quota": 500000, "max_quota": 750000 }
}
```

说明：

- 普通用户签到比会员少，用于鼓励充值和长期使用。
- Standard 优回到原先普通签到体验附近，作为第一层会员福利。
- Pro优、Super优、Ultra优逐级提高，但仍保持随机波动，避免每天固定到账。
- 如果后台没有命中某个分组配置，会回退到默认签到范围。
- 签到范围是随机区间，不是固定到账金额；同一等级每天金额不同是正常现象。

## AI 小助手回答边界

可以回答：

- 当前会员等级有哪些、怎么升级、各等级大概有什么权益。
- 为什么不同用户签到金额不同。
- 为什么充值后没有马上变成会员。
- Ultra优能使用哪些 Claude 渠道或模型。
- 折扣权益的大致口径。

不要承诺：

- “一定会自动升级”或“今天一定升级”。
- 人工补签、人工补余额、人工修改会员等级。
- 退款、赔付、赠送额度。
- 任何未在用户当前模型列表或后台权限中开放的模型。
- 绕过分组限制、折扣限制或渠道限制。

遇到不确定情况时，统一建议用户提供账号、充值记录、消耗记录、截图和大概时间，交由管理员核查。

## 常见问答

### Q1：我为什么还是普通用户？

普通用户是默认身份。会员升级通常需要同时满足两个条件：有过充值记录，并且累计余额消耗达到对应门槛。

如果只是刚充值、还没有消耗到门槛，一般不会立即升级。若你确认已经达到门槛但页面仍显示普通用户，请带上账号、充值记录、消耗截图和大概时间联系管理员核查。

### Q2：Standard 优怎么获得？

Standard 优是第一层会员。通常需要有过充值记录，并且累计余额消耗满 10。

Standard 优的主要福利是比普通用户更高的每日签到奖励，展示名请写作 `Standard 优`。

### Q3：Pro优、Super优、Ultra优怎么升级？

需要有过充值记录，并达到对应累计余额消耗门槛：

- Pro优：累计消耗满 50。
- Super优：累计消耗满 200。
- Ultra优：累计消耗满 500。

达到门槛后，系统或管理员侧会按实际规则调整分组。页面显示以账号当前分组为准。

### Q4：为什么我的签到金额和别人不一样？

签到奖励按会员等级分区间随机发放。普通用户、Standard 优、Pro优、Super优、Ultra优的随机区间不同；同一等级每天也可能出现不同金额。

只要到账金额在当前等级配置区间内，就属于正常波动。

### Q5：为什么今天签到比昨天少？

签到不是固定金额，而是在当前等级区间内随机。今天比昨天少，或明天比今天多，都可能是正常情况。

如果你认为到账金额明显不符合等级区间，请带上签到截图、账号和签到时间联系管理员核查。

### Q6：会员折扣是什么？

当前权益口径：

- Pro优：全场 98 折。
- Super优：全场 95 折。
- Ultra优：全场 92 折。

实际扣费和折扣生效以后台计费配置、模型配置和账单记录为准。

### Q7：Ultra优能使用哪些 Claude？

Ultra优可使用 AWS、Anthropic API、MAX 反代等官方或高阶渠道的 Claude 模型。

具体可用模型会随渠道状态、后台分组权限和模型列表变化而变化。请以账号当前能看到和能调用的模型为准。

### Q8：充值后多久会升级？

升级不是只看充值金额，还要看累计余额消耗。满足“有充值记录 + 达到消耗门槛”后，通常需要等待系统同步或管理员侧分组更新。

如果长时间没有变化，请提供账号、充值记录、消耗记录和大概时间让管理员核查。

### Q9：能不能帮我手动改成更高会员？

AI 小助手不能直接修改会员等级。会员等级以账号分组和后台规则为准。

如果你认为等级识别有误，请提供充值和消耗证明，由管理员核查后处理。

### Q10：能不能补签或补发签到奖励？

AI 小助手不能承诺补签或补发。签到是否补发由管理员根据后台记录判断。

如果是系统异常导致无法签到，请提供错误截图、账号和发生时间。

## 管理员参考

### 会员铭牌配置

新 UI 后台页面 `/admin/page-config` 支持配置会员铭牌文案，包括：

- 普通用户
- Standard 优
- Pro优
- Super优
- Ultra优

配置会保存到 `ui_page_config.membership_badges`。用户侧通过 `GET /api/ui/page-config` 读取并覆盖默认文案。

字段限制：

- 名称不超过 30 个字符。
- 短名不超过 20 个字符。
- 描述不超过 80 个字符。

### 签到分组配置

签到配置项：

- `checkin_setting.min_quota`：默认最小签到额度。
- `checkin_setting.max_quota`：默认最大签到额度。
- `checkin_setting.group_quotas`：按用户分组覆盖签到额度。

`group_quotas` 支持精确分组名，也支持会员语义识别：

- `standard` / `standard优` / `stand`
- `pro` / `pro优`
- `super` / `super优` / `spuer`
- `ultra` / `ultra优`

如果配置项为空、分组未命中或加载失败，会使用全局默认签到范围。

### 推荐给 AI 小助手的简短总答复

会员等级按账号当前分组展示，主要分为普通用户、Standard 优、Pro优、Super优、Ultra优。升级通常需要有过充值记录，并达到对应累计消耗门槛：Standard 优满 10，Pro优满 50，Super优满 200，Ultra优满 500。每日签到按会员等级在区间内随机发放，所以金额每天可能不同。Pro优享受全场 98 折，Super优 95 折，Ultra优 92 折；Ultra优还可使用 AWS、Anthropic API、MAX 反代等官方或高阶渠道的 Claude 模型，具体以当前后台权限和模型列表为准。
